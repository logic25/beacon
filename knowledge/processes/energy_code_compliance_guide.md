# Energy Code Compliance (NYCECC) Submission Guide

## Overview

The New York City Energy Conservation Code (NYCECC) is NYC's adopted building energy code governing energy efficiency requirements for buildings. This guide covers the regulatory framework, compliance pathways, filing requirements, and common resolution strategies for energy code submissions through the Department of Buildings (DOB).

## What is NYCECC?

The New York City Energy Conservation Code is NYC's adopted energy efficiency standard, based on the ASHRAE (American Society of Heating, Refrigerating and Air-Conditioning Engineers) 90.1 standard.

### Regulatory Basis

- **Local Law 29 of 2009:** Established NYC energy code authority and integration into NYC Building Code
- **ASHRAE 90.1 Standard:** Nationally recognized energy efficiency standard
- **NYC Amendments:** Additional NYC-specific requirements on top of ASHRAE 90.1
- **Alignment with LL97 and LL154:** Energy code compliance coordinates with broader NYC emissions and electrification mandates

### Code Evolution

- **2020 NYCECC:** Based on ASHRAE 90.1-2016 with NYC amendments (current primary standard as of 2025)
- **Future updates:** ASHRAE 90.1-2019 and 90.1-2022 versions under consideration for adoption [VERIFY current adoption timeline]
- **Interim compliance:** Projects following earlier editions may be grandfathered if filed before amendment effective date

### Code Applicability Rules

- **New construction:** Must comply with 2020 NYCECC (ASHRAE 90.1-2016)
- **Existing buildings with major alterations:** Must comply with current code for altered systems
- **Change of use:** May trigger code compliance for affected building systems
- **Renovation scope:** Depends on percentage of building renovation (see When Energy Code Compliance is Required section)

**Important:** The code year is locked in at filing. Contractors cannot request different code year after DOB review begins.

## When Energy Code Compliance is Required

Energy code compliance submissions are required in these scenarios:

### New Building Construction

**Requirement:** All new buildings, regardless of size or type, must comply with NYCECC

**Scope:**
- Complete building design (HVAC, lighting, insulation, air sealing, hot water, etc.)
- All building systems
- No exemptions based on size or use

**ECC filing:** Must be included with initial building permit filing in DOB NOW

### Major Alterations

**Definition:** Alterations where the cost exceeds 25% of building replacement cost [VERIFY percentage threshold]

**Requirement:** All altered or replaced building systems must comply with NYCECC

**Scope examples:**
- HVAC system replacement → new system must meet code
- Lighting retrofit affecting >50% of floor area → new lighting must meet code
- Envelope work (windows, insulation, roofing) → new components must meet code

**Code scope:** Only altered systems; existing systems not being replaced may be exempt [VERIFY if existing systems must be assessed]

**ECC filing:** Submit with building permit for the alteration work

### Change of Use

**Requirement:** If building use changes to a different category, energy code compliance review triggered

**Use categories:**
- Residential (Group R)
- Commercial office (Group B)
- Retail/mercantile (Group M)
- Assembly/public (Group A)
- Industrial (Group F/H)
- Other (specific categories)

**Trigger threshold:** Change from one major category to another (e.g., industrial warehouse to commercial office)

**Code scope:** May apply to entire building or affected portion depending on change scope [VERIFY]

**ECC filing:** Required even if no construction work; filed as variance or special filing

### Renovation/Modernization of Significant Scope

**Definition:** Building work exceeding approximately 20-25% of wall area, roof area, or floor area [VERIFY exact threshold]

**Requirement:** Comprehensive energy audit; compliance demonstrated for affected building systems

**Code scope:** Building systems affected by renovation; sometimes extends to building envelope even if not being directly altered

**ECC filing:** Required as part of permit package

These project types are generally exempt:

- **Limited alterations:** Work affecting <20% of affected system or building area [VERIFY threshold]
- **Like-for-like replacements:** System replacement with identical equipment (e.g., boiler to same-model boiler) may be exempt in some cases [VERIFY current policy]
- **Emergency repairs:** Critical repairs to maintain building operation (with post-completion documentation)
- **Tenant fit-out:** Interior finishes only (no building system work) in buildings with existing ECC approval [VERIFY if depends on base building approval]
- **Historic buildings:** Buildings on NYC Landmarks Preservation Commission list may have modified requirements [VERIFY LPC coordination]

## Compliance Paths

Projects must demonstrate energy code compliance through one of three primary pathways:

### Path 1: Prescriptive Compliance

**Definition:** Meeting code by following prescriptive requirements (specific standards for insulation R-values, HVAC efficiency ratings, lighting power density, etc.)

**How it works:**
- Design follows code-specified minimum values for each building component
- No calculations or trade-offs
- Most straightforward path for smaller projects
- Designer documents compliance with code sections

**Best for:**
- Small buildings or building portions
- Standard construction methods
- Projects with no unusual building features

**Deliverable to DOB:**
- ECC filing form (ECC-1) [VERIFY form name]
- Design narrative describing prescriptive compliance
- Equipment schedules with rated efficiencies (SEER, EER, HSPF, lighting specifications)
- Insulation specification sheets
- Window/door thermal performance documentation (U-values, SHGC)

**Timeline:**
- Preparation: 1-2 weeks (for design team to compile existing specifications)
- DOB review: 3-5 business days
- Objections/revisions: Variable (see Common Energy Code Objections section)

**Cost/effort:** Moderate; primarily documentation compilation

### Path 2: Performance Compliance (COMcheck / REScheck)

**Definition:** Meeting code through energy modeling tools (COMcheck for commercial, REScheck for residential) that demonstrate equivalent energy performance

**How it works:**
- Building design modeled in COMcheck or REScheck software (provided free by DOE/IECC)
- Proposed design compared to baseline code building
- If proposed design performs as well as or better than code baseline, compliance achieved
- Allows trade-offs (e.g., lower insulation if HVAC system more efficient)

**Best for:**
- Commercial buildings with mixed strategies (some components above code, some below)
- Residential buildings with efficient mechanical systems
- Projects seeking design flexibility without full energy modeling

**Deliverable to DOB:**
- ECC filing form
- COMcheck/REScheck output report (compliance certification from software)
- Building envelope specifications (window U-values, insulation R-values, air barrier description)
- HVAC system specifications (efficiency ratings)
- Lighting power and controls specification
- Service hot water system details
- Narrative explaining compliance approach

**Timeline:**
- Preparation: 2-4 weeks (for MEP engineer to model building in software, gather specifications)
- DOB review: 3-5 business days
- Objections/revisions: Variable (may require model re-run if conditions change)

**Cost/effort:** Moderate; requires MEP engineer with COMcheck/REScheck expertise

**Tools:**
- **COMcheck:** https://www.energycodes.gov/comcheck (free tool from U.S. DOE)
- **REScheck:** https://www.energycodes.gov/rescheck (free tool from U.S. DOE)
- **Cost:** Free; some consulting firms charge for preparation services

**Important notes:**
- COMcheck/REScheck are pre-defined tools and compliance is typically accepted by DOB without objection
- Models must use correct IECC/ASHRAE edition (2020 NYCECC = ASHRAE 90.1-2016)
- Trade-offs are allowed as long as overall building performance meets baseline [VERIFY if NYC allows trade-offs or if all components must meet minimums]

### Path 3: Energy Modeling (Simulation-Based Compliance)

**Definition:** Demonstrating compliance through detailed whole-building energy simulation (using tools like eQuEST, EnergyPlus, OpenStudio, or LEED simulation software)

**How it works:**
- Detailed building energy model created (all systems, occupancy, schedules, controls)
- Proposed design modeled with actual equipment and systems
- Model results compared to code baseline building
- If proposed design performs at code baseline level (or better), compliance achieved
- Allows for complex strategies, renewable energy, district systems, etc.

**Best for:**
- Large complex buildings
- Buildings with renewable energy systems
- District energy connections
- Advanced controls or unusual mechanical systems
- Projects seeking high-performance certification (LEED, Passive House, etc.)
- Buildings with significant baseline performance challenges where standard paths don't work

**Deliverable to DOB:**
- ECC filing form
- Energy modeling report (typically 20-50 pages) including:
  - Building description and assumptions
  - System descriptions and efficiencies
  - Envelope specifications
  - Operating schedules
  - Weather data and modeling software used
  - Annual energy consumption results
  - Comparison to code baseline
  - Verification that proposed design meets or exceeds baseline performance
- Building envelope specifications
- Mechanical, electrical, and plumbing drawings
- Controls narrative
- Renewable energy system specifications (if included)

**Timeline:**
- Preparation: 4-12 weeks (requires detailed design and specialized expertise)
- DOB review: 7-10 business days (more complex, longer review)
- Objections/revisions: Common; may require model re-runs and sensitivity analyses

**Cost/effort:** Highest; requires specialized energy modeler and significant MEP design detail

**Modeling software:**
- **eQuEST:** Free tool; complex interface
- **EnergyPlus:** Free, detailed; technically complex
- **OpenStudio:** Free; user-friendly
- **LEED/USGBC tools:** Often used for certified projects
- **Commercial software:** HAP, TRACE, DesignBuilder (licensed, $2-10K annually)

**Important notes:**
- Modeling approach must be DOB-approved; some tools/consultants have established track records
- More flexibility for trade-offs and advanced strategies
- Requires more rigorous documentation
- Model must account for NYC climate and utility costs
- More likely to receive DOB objections requiring clarification [VERIFY objection rates]

## ECC Filing Requirements in DOB NOW

All energy code compliance submissions must be filed electronically through DOB NOW.

### System Access

- **DOB NOW:** www1.nyc.gov/html/dob/html/dob_now/dobhome.shtml
- **Account creation:** Register with NYC Department of Buildings credentials
- **Authorization:** Design professional (architect or engineer) or contractor can file with proper licensure

### Filing Steps

1. **Log in to DOB NOW**
   - Use NYC credentials (same as other DOB filings)

2. **Select project type**
   - Navigate to "Building Permits" → "New Filing"
   - Or "Amendments" if modifying existing permit

3. **Indicate energy code compliance**
   - Check "Energy Code Compliance Certification" or similar option [VERIFY exact DOB NOW field name]
   - Note which ECC path being followed (prescriptive, COMcheck/REScheck, energy modeling)

4. **Provide building information**
   - Building classification (use group)
   - Building area (square feet)
   - Number of stories
   - Building climate zone (NYC = Zone 6A per ASHRAE)

5. **Energy design professional information**
   - Name and license number of PE or architect responsible for energy code compliance
   - Contact information

6. **Compliance approach description**
   - Narrative (500-1000 words) explaining compliance path
   - Key efficiency strategies employed
   - Any innovative features or advanced systems

7. **Upload ECC documentation** (see Required Forms and Documentation section)

8. **Pay fee** if applicable [VERIFY if ECC filing has separate fee or included in permit fee]

9. **Submit** for DOB review

10. **Track status** in DOB NOW (email notifications sent during review)

### Document Upload Requirements

- **Format:** PDF (preferred); images JPG/PNG acceptable
- **Resolution:** Minimum 200 dpi for legibility
- **Page naming:** Each document clearly labeled (e.g., "ECC_Report_05.pdf")
- **Complete submissions:** All required documents must be uploaded; incomplete submissions extend review 5-10 days
- **File size:** Individual files should be <25 MB; if larger, split into multiple files

## Progress Inspections for Energy Components

The DOB conducts progress inspections to verify that installed building systems match submitted ECC specifications.

### When Progress Inspections Occur

Energy component inspections typically occur:

1. **After envelope systems are substantially in place** (insulation installed, air barriers sealed)
2. **Before mechanical systems are enclosed** (ductwork, pipes visible for inspection)
3. **At fixture/equipment installation stage** (HVAC units, lighting fixtures, controls)
4. **Before final inspection**

Exact timing depends on project schedule and inspector coordination.

### Components Inspected

**Building Envelope:**
- Insulation installation: location, R-value compliance (checked against specification), continuity
- Air barrier: sealing methods, tape used, continuity of barrier
- Window installation: U-value compliance, proper sealing, SHGC compliance
- Door/penetration sealing: continuity of envelope

**HVAC Systems:**
- Equipment: nameplate verification, model number matches specifications
- Ductwork insulation: R-value, sealing of joints
- Thermostatic controls: presence and location, commissioning (if required)
- Ventilation dampers/controls: proper operation

**Lighting Systems:**
- Fixture count and location: matches specification
- Lamp/ballast type: efficiency ratings verified
- Controls: occupancy sensors, dimming, daylight sensors (if specified)
- Power density calculation: verified against specification

**Service Hot Water:**
- Water heater: model verification, efficiency rating
- Pipe insulation: material, thickness, coverage

**Renewable Energy Systems** (if installed):
- Equipment installation per specification
- Electrical connections (for PV systems)
- Performance monitoring/commissioning

### Inspector Authority

DOB inspectors conducting energy component inspections can:

- **Demand access:** Enter all construction areas to inspect building systems
- **Require submittals:** Ask for equipment cut sheets, material certifications
- **Fail inspection:** If installed components do not match ECC submission
- **Request changes:** May require reinstallation if non-compliant (e.g., insufficient insulation)
- **Approve compliance:** Sign off on energy components once verified

### Failing an Energy Inspection

If inspection is failed:

1. **Reason documented:** Inspector notes specific non-compliance (e.g., "Insulation installed is R-15, specification calls for R-21")
2. **Contractor notification:** Inspector notifies contractor/superintendent of deficiency
3. **Correction deadline:** Contractor must correct deficiency and request re-inspection (usually within 10-15 days [VERIFY])
4. **Re-inspection:** DOB returns to verify correction
5. **Cost:** Corrections are contractor responsibility; re-inspection fees may apply [VERIFY if re-inspection fee applies]

### Commissioning/Performance Testing

**Mechanical commissioning** may be required by ECC, particularly for complex HVAC systems:

- **HVAC testing:** Verify equipment operation, airflow, temperature control
- **Controls testing:** Verify building automation system functions per specification
- **Documentation:** Commissioning report submitted to DOB
- **Cost:** Contractor responsible; can be significant for large buildings [VERIFY typical cost range]

**Energy Star certification** or similar performance benchmarking may be required for some buildings [VERIFY if NYC requires post-occupancy performance verification].

## Common Energy Code Objections and Resolution

The DOB regularly identifies deficiencies or ambiguities in ECC submissions. Understanding common objections and resolution approaches helps expedite approval.

### Objection Category 1: Incomplete or Missing Specifications

**Common objections:**
- "Equipment model numbers not provided for HVAC units"
- "Window U-value and SHGC not specified"
- "Insulation R-values not documented"
- "Control system strategy not clearly described"

**Root cause:** Design team has not finalized specifications, or documentation incomplete

**Resolution:**
1. Obtain missing specifications from MEP engineers and product manufacturers
2. Document in memo format (1-2 pages) with:
   - Specification of what was missing
   - Exact values/model numbers
   - Manufacturer cut sheets attached
   - Engineer certification that specifications meet code
3. Upload to DOB NOW as "Clarification Submission"
4. Resubmit within 5-7 business days
5. Typical resolution: 2-3 business days after resubmission

### Objection Category 2: Code Baseline Calculations Incorrect

**Common objections:**
- "Lighting Power Density (LPD) calculation exceeds code maximum" [e.g., calculated as 1.5 W/sq ft but code allows 1.2 W/sq ft]
- "HVAC sizing does not match code climate zone requirements"
- "Window-to-wall ratio exceeds code limits"

**Root cause:** Mathematical error in ECC document, or misunderstanding of code requirements

**Resolution:**
1. Engage qualified energy engineer or MEP designer to recalculate
2. Identify the error (calculation method, input assumption, code section misinterpreted)
3. Prepare corrected submittal with:
   - Revised calculations or input values
   - Detailed explanation of error and correction
   - Reference to applicable code sections
   - If significant change, may need design adjustment (e.g., reduce LPD by adding controls, or adjust window design)
4. Submit corrected ECC documentation
5. Typical resolution: 3-5 business days if calculation corrected; longer if design adjustment required

### Objection Category 3: Inadequate Documentation

**Common objections:**
- "Energy modeling report does not adequately justify assumptions"
- "COMcheck baseline building not clearly documented"
- "Building classification ambiguous; does not align with submitted building code use group"

**Root cause:** Technical report written for internal use; insufficient detail for DOB review

**Resolution:**
1. Prepare supplementary documentation (5-10 pages) including:
   - Clear statement of building classification and use group
   - Detailed assumptions (occupancy schedules, operating hours, plug loads)
   - Baseline building description (how baseline differs from proposed design)
   - Clarification of any ambiguous terms or calculations
2. Provide references to code sections cited
3. Submit as supplementary memo with original ECC documents
4. Typical resolution: 2-3 business days

### Objection Category 4: Discrepancies Between ECC and Building Design

**Common objections:**
- "ECC specifies triple-glazed windows (U-0.28), but window schedule on architectural drawings shows double-glazed (U-0.35)"
- "ECC indicates occupied zone conditioning, but floor plan shows open warehouse without HVAC distribution"
- "Lighting specification in ECC does not match lighting plan on electrical drawings"

**Root cause:** ECC prepared in parallel with design development; design changed but ECC not updated

**Resolution:**
1. **Best approach:** Coordinate between energy engineer and design team to align documents
   - Determine whether ECC or design is correct
   - If design is preferred, revise ECC to match
   - If ECC specifications are required, update design documents
2. **Prepare clarification memo** explaining:
   - Which document is controlling
   - How discrepancy will be resolved
   - If design change needed, attach revised drawings
3. **Resubmit aligned documents** (both revised ECC and design documents)
4. Typical resolution: 3-7 business days (if minor clarification); 1-2 weeks (if design changes needed)

### Objection Category 5: Compliance Path Not Acceptable for Building Type

**Common objections:**
- "Prescriptive path not acceptable for buildings >25,000 sq ft; must use COMcheck or energy modeling"
- "Standard COMcheck insufficient for complex renovation; energy modeling required"
- "IECC 2016 baseline does not align with ASHRAE 90.1-2016; must use correct baseline"

**Root cause:** DOB may require specific compliance approach for complex buildings; compliance path selected does not meet DOB expectations

**Resolution:**
1. Obtain written clarification from DOB on why selected path was rejected
2. If required to change path:
   - **Prescriptive to COMcheck:** 2-4 weeks to prepare COMcheck model
   - **Prescriptive/COMcheck to energy modeling:** 6-12 weeks to prepare detailed energy model
3. Engage qualified professional for new approach
4. Resubmit complete new ECC package under required path
5. Typical timeline: See timelines under Compliance Paths section

### Objection Category 6: Non-Compliance with Specific Code Provisions

**Common objections:**
- "Proposed HVAC efficiency does not meet ASHRAE 90.1 Table [X] requirements"
- "Boiler efficiency rating below code minimum"
- "Insulation not continuous; gaps exceed code allowance"

**Root cause:** Designed equipment/systems do not meet actual code requirements (not a documentation issue; design deficiency)

**Resolution:**
1. **Upgrade design:** Select higher-efficiency equipment, add insulation, modify strategy
   - HVAC system with higher SEER/HSPF rating
   - Insulation additional thickness
   - Boiler with higher thermal efficiency
2. **Request code variance:** If physical constraints prevent compliance, petition for variance [VERIFY DOB variance process for energy code]
3. **Alternative compliance strategy:** Demonstrate compliance through different pathway (trade-offs through COMcheck or energy modeling)
4. **Resubmit revised ECC:** With upgraded equipment/systems or detailed variance justification
5. Typical resolution: 1-2 weeks if design upgrade straightforward; 2-4 weeks if variance needed

### Objection Resolution Strategy

**General best practices for objection resolution:**

1. **Read objection carefully:** DOB objection letter will cite specific code section and explain deficiency
2. **Engage design team:** Contact MEP engineer and architect to understand and address
3. **Prepare thorough response:** Don't submit quick fixes; provide complete documentation addressing root cause
4. **Reference code:** Cite ASHRAE 90.1 sections and NYC amendments in response
5. **Timeline management:** Plan for multiple rounds of objections (average 2-3 rounds for complex buildings [VERIFY typical number])
6. **Professional certification:** All responses sealed by licensed PE or architect
7. **Proactive communication:** Contact DOB ECC reviewer if complex issues; ask for guidance before resubmitting

**Objection response timelines:**
- Objection issued: Project placed on hold
- Response deadline: Typically 30-45 days [VERIFY deadline]
- Review of response: 5-10 business days
- If additional objections: Another 30-45 day cycle

## Relationship to NYC LL97 (Building Emissions)

### Overview of LL97

**Local Law 97 of 2019** (enacted as part of NYC's Climate Mobilization Act) mandates energy efficiency and greenhouse gas emissions reductions for buildings. ECC compliance supports but does not automatically satisfy LL97 requirements.

### LL97 Applicability

**Buildings subject to LL97:**
- Buildings >25,000 sq ft built before 2000 [VERIFY date threshold and square footage]
- Buildings >25,000 sq ft with major renovations after 2020
- All buildings after 2030 [VERIFY future phase-in dates]

**Buildings with new construction/major work:**
- If project involves major alteration triggering ECC, consider LL97 implications
- New buildings naturally comply with LL97 if they meet current energy code
- Renovations should design to LL97 compliance path even if not currently mandated

### Relationship Between ECC and LL97

**ECC compliance is a foundation for LL97 compliance:**
- Buildings meeting 2020 NYCECC will have lower baseline energy use
- Lower baseline helps meet LL97 emissions targets in 2024-2029 and 2030+ phases

**LL97 goes beyond ECC:**
- ECC sets minimum efficiency standards; LL97 sets actual emissions caps (carbon intensity limits)
- Complying with ECC does not guarantee LL97 compliance
- Existing building must actively reduce operations-phase emissions

**For projects with both ECC and LL97 requirements:**
- Exceed minimum ECC standards where feasible
- Include high-efficiency systems (beyond code minimum)
- Consider renewable energy to offset grid electricity
- Model lifecycle emissions, not just code compliance

### LL97 Scoring System

**Compliance measured by:**
- **kgCO2e/sq ft (carbon intensity):** Buildings must report actual energy use and calculate emissions
- **Emissions caps:** Stricter in 2030 phase than 2024-2029 phase
- **Flexibility mechanisms:** Carbon credits, renewable energy, building electrification offset emissions

**For new construction/major renovation:**
- Design with LL97 compliance in mind
- Specify high-efficiency systems, electrification, renewable energy
- Submit energy model showing LL97 compliance path
- ECC submission should reference LL97 alignment

## Relationship to NYC LL154 (Electrification)

### Overview of LL154

**Local Law 154 of 2021** requires buildings to achieve full electrification (no fossil fuel use) by specific phases, starting with new buildings and major alterations.

### LL154 Requirements

**New buildings (filed permit 2027+ or construction starts after 2027):** [VERIFY dates]
- Full electrification required
- No natural gas or other fossil fuels
- All heating, hot water, and cooking must be electric

**Major alterations (to any building):**
- Fossil fuel systems being replaced must convert to electric
- Fossil fuel infrastructure must be removed or disconnected
- Heat pump systems and electric resistance heating required

**Timeline:**
- Likely to be phased in 2027-2035+ [VERIFY current schedule; law may be updated]

### LL154 Integration with ECC

**Electrification supports energy code compliance:**
- Electric heat pump systems are high-efficiency (COP 3-4 typical vs. 0.85 for natural gas furnace)
- Heat pumps meet or exceed ASHRAE 90.1 efficiency requirements
- Eliminates combustion losses (>10% heat loss with gas furnaces)

**For projects with ECC + LL154 requirements:**
- Specify electric heat pumps (heating, cooling, hot water)
- Size heat pump systems appropriately for climate zone
- Include electric resistance backup (if heat pump alone insufficient)
- Design for high insulation levels to support efficient heat pump performance
- Consider demand-response-ready equipment and controls

**ECC + LL154 benefits:**
- Lower operating costs (electricity cheaper than gas in many cases)
- Improved air quality (no combustion emissions)
- Better path to LL97 emissions compliance
- Future-proofed building (laws will only become stricter)

### Solar PV System Design (If Installing Solar)

If solar PV system is being designed/installed as part of project:

**ECC considerations:**

- Annual energy output (kWh/year) and offset to building consumption
- May count toward energy modeling compliance (reduces building's grid electricity demand)

### Typical ECC Review Timeline by Compliance Path

#### Prescriptive Path Timeline

| Phase | Duration | Notes |
|---|---|---|
| Design/specification preparation | 1-2 weeks | MEP engineer compiles specifications, creates schedule |
| ECC document preparation | 1 week | Document compiled, sealed, uploaded |

| Objections (if any) | 7-15 business days | Contractor corrects deficiencies, resubmits |
| DOB final review | 2-3 business days | DOB approves if objections resolved |
| **Total (best case):** | **2-3 weeks** | No objections, straightforward specifications |
| **Total (typical case):** | **4-6 weeks** | 1-2 rounds of objections |

#### COMcheck/REScheck Path Timeline

| Phase | Duration | Notes |
|---|---|---|
| Design finalization | 1-2 weeks | MEP design substantially complete |
| COMcheck/REScheck model preparation | 2-4 weeks | Engineer builds model, gathers all specifications |
| ECC document preparation | 1 week | Report compiled from COMcheck output, sealed |
| DOB initial review | 3-5 business days | Review for completeness, model accuracy |
| Objections (if any) | 10-20 business days | Model may require adjustment, re-runs, re-submissions |
| DOB final review | 3-5 business days | Final approval |
| **Total (best case):** | **4-5 weeks** | COMcheck/REScheck straightforward, minimal objections |
| **Total (typical case):** | **6-8 weeks** | 1-2 rounds of objections, model adjustments |

#### Energy Modeling Path Timeline

| Phase | Duration | Notes |
|---|---|---|
| Detailed design development | 2-4 weeks | MEP design finalized, all equipment specified |
| Energy model creation | 4-8 weeks | Modeler builds detailed model, validates against design |

### Factors Affecting Timeline

**Delays:**
- Incomplete initial submission (add 5-10 days per round)
- Design changes during review (add 3-7 days per change)
- Complex building requiring escalation (add 7-14 days)
- Third-party consultant availability (model, engineer, noise specialist, etc.)
- Multiple rounds of objections (add 7-15 days per round)

**Accelerations:**
- Early engagement of energy engineer during design phase
- Complete specification during design (not during ECC preparation)
- Using straightforward compliance path (prescriptive for simple buildings)
- Proactive objection resolution (respond within 5 days, not waiting until deadline)
- Expediter coordination with DOB

### Project Milestones

**For planning construction schedule:**

1. **Energy code strategy selection:** Month 1 of design (determine prescriptive vs. COMcheck vs. modeling)
2. **MEP system selection:** Month 2-3 of design (all equipment/systems specified for ECC compliance)
3. **ECC documentation preparation:** Month 3-4 of design (document ready to submit with building permit)
4. **ECC filing with DOB:** Concurrent with building permit filing (Month 4-5)
5. **ECC approval:** Expected Month 5-7 (depending on path, objections)
6. **Construction mobilization:** Typically occurs after building permit approval (which may depend on ECC approval)
7. **Progress inspections:** During construction, as systems installed (Month 8-14+ depending on project duration)

**Critical path:** ECC approval should not delay overall project permitting schedule; plan accordingly.

## Best Practices Summary

### For Project Teams Navigating ECC

1. **Engage energy engineer early:** Don't wait until permit preparation; involve in design development
2. **Select compliance path strategically:** Match building complexity to appropriate path (don't over-engineer simple buildings; don't under-document complex ones)
3. **Coordinate across disciplines:** Architectural, MEP, and energy engineering must align on building design
4. **Document thoroughly:** Complete specifications, clear narratives, equipment cut sheets all essential
5. **Anticipate DOB expectations:** Familiarize team with common objections; proactively address in initial submission
6. **Plan for timeline:** Factor 6-12 weeks into schedule for ECC review and potential revisions
7. **Budget appropriately:** Professional modeling, consulting fees, and potential design changes are normal costs
8. **Stay engaged with DOB:** Answer objections promptly, provide requested clarifications quickly

### Design Recommendations for LL97/LL154 Alignment

Even for projects where LL97/LL154 are not immediately applicable, design with future compliance in mind:

1. **Exceed code minimums:** Add extra insulation, higher-efficiency equipment, renewable-ready infrastructure
2. **Specify electrification:** Even if not required, design for future fossil fuel elimination
3. **Include solar readiness:** Pre-design for potential PV system
4. **Advanced controls:** Design for sophisticated monitoring and control systems
5. **Resilience:** Consider climate resilience, flood protection, backup power systems
