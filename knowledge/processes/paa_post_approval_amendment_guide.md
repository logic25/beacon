# PAA (Post Approval Amendment) Filing Guide

## 1. What is a PAA?

A **Post Approval Amendment (PAA)** is a formal amendment to an already-approved DOB application/permit. It allows applicants to modify the scope of work, design, or plans **after** the DOB has issued approval (AC or Initial Approval status) but **before** or **during** construction.

A PAA does NOT restart the review process from scratch—it amends the existing approved application while maintaining the original approval number and application history.

**Key Point:** PAA ≠ Place of Assembly (PA/PACO). PAA is specifically for amending approved applications.

---

## 2. PAA vs. New Application: When to Use PAA

### Use PAA When:
- The original application is already approved (has AC or Initial Approval)
- Changes are relatively minor to moderate in scope
- You want to maintain the original approval number and timeline
- The change can be reasonably documented and justified
- Professional certification can be obtained for the modified scope

### Use New Application When:
- The change is so substantial it fundamentally alters the project (e.g., change from residential to commercial occupancy)
- Multiple major structural systems are redesigned
- The original application was rejected and you're filing a different approach
- The property characteristics or zoning status has changed
- You need to reset the review timeline
- Changes exceed the scope of the original permit class

---

## 3. Minor PAA vs. Major PAA

### Minor PAA
**Scope:** Smaller adjustments that don't significantly alter structural integrity, safety, or code compliance.

**Examples:**
- Field condition changes requiring minor layout adjustments
- Material substitutions (e.g., different window type, same performance level)
- Minor dimensional changes (e.g., moving a non-structural wall 6 inches)
- Correction of minor drafting errors on approved plans
- Addition of minor work not originally included
- Contractor substitution or equipment vendor changes
- Small changes to mechanical/electrical systems within code limits

**Processing:** Typically faster review (5-10 days), often single examiner review, standard service tier applies.

**Fee:** Standard PAA tier ($750 / 2 hours) [VERIFY current DOB rates]

---

### Major PAA
**Scope:** Significant changes that could affect structural design, egress, occupancy, safety systems, or code compliance.

**Examples:**
- Structural modifications (columns moved, beams resized, load paths changed)
- Egress changes (door relocations, stairway modifications, exit count changes)
- Occupancy category changes (residential to commercial, office to assembly)
- Fire safety system redesign
- Significant mechanical/electrical/plumbing system overhauls
- Changes to required department (BIS, FDNY, etc.) coordination
- Substantial scope expansion beyond original filing

**Processing:** More rigorous review, multiple examiners, potential objections, plan corrections likely. Can take 15-30 days or longer.

**Risk:** DOB may determine change is too significant and require **new application filing**.

**Fee:** Large change PAA tier ($1,750 / 4 hours) [VERIFY current DOB rates]

**Note:** The distinction between Minor and Major PAA determines both fee tier and review complexity. When in doubt, assume Major PAA to avoid rejections.

## 4. PAA Filing Process: Step-by-Step in DOB NOW

### Step 1: Verify Original Application Status
- Log into DOB NOW
- Locate the original Application/Job ID
- Confirm status is **"Approved"** or **"AC Issued"** (not rejected or pending)
- Review the **Approved Plans** to understand what was originally approved

### Step 2: Prepare Documentation Package

Gather the following before filing:

1. **Amended Plans** (see Section 5 for requirements)
2. **Professional Certification Letter** (see Section 7)
3. **Explanation Letter** describing:
   - What is being changed
   - Why the change is necessary
   - How it complies with code
   - Impact on original approval scope
4. **Comparison Document** (recommended):
   - Marked-up plans showing changes (red lines) vs. original approved plans
   - Side-by-side comparison table
5. **Any Supporting Documentation**:
   - Field condition reports (if field change driven)
   - Building owner authorization letter
   - Revised specifications (if applicable)

### Step 3: Initiate PAA in DOB NOW

1. Navigate to the original Job ID in DOB NOW
2. Select **"File Amendment"** or **"Create Amendment"** [VERIFY exact button label in current DOB NOW version]
3. Choose **"Post Approval Amendment"** as amendment type
4. Select **PAA Classification**:
   - Minor PAA (if straightforward)
   - Major PAA (if complex or uncertain)
5. Enter application details:
   - Job ID (auto-populated)
   - Filing fee (system calculates based on tier)
   - Applicant information

### Step 4: Upload Documents

Upload documents in this order:
1. Amended Plans (primary document)
2. Professional Certification Letter
3. Explanation Letter
4. Marked-up Comparison Plans
5. Supporting Documentation

**File Format Requirements:** [VERIFY]
- Plans: PDF (preferred) or acceptable CAD formats
- Letters: PDF
- File names should be descriptive (e.g., "Amended_Plans_Foundation_Changes_PAA.pdf")

### Step 5: Certify and Submit

- Certify that all information is true and accurate
- Review estimated fees and timeline
- **Submit for Review**
- Obtain **PAA Confirmation Number**
- Screenshot or save confirmation for records

### Step 6: Track Review Status

- DOB NOW displays status updates in real-time
- Monitor for:
  - **Examiner Assigned**
  - **In Review**
  - **Objections Issued** (if any)
  - **Approved** / **Approved With Corrections**

## 5. Required PAA Documents

### Amended Plans
**Requirements:**
- Show **all modified elements** clearly
- Include **red-lined or color-coded changes** from original approved plans
- Must be signed and sealed by licensed PE/RA (see Section 7)
- Include **plan legend** explaining what was changed
- Show original dimensions/notes in lighter color with new versions bold/highlighted
- Sheet numbers should match original approved set for easy reference
- Include **revision mark/date** (e.g., "Revision 1 - January 2025")
- All notes referencing original approval number ("see approved plans dated...")

**What Must Be Included:**
- All sheets affected by the change (even if minor revisions)
- Typical sheets: architectural, structural (if applicable), mechanical, electrical, etc.
- A **"Changes Summary Sheet"** listing all amendments

### Professional Certification Letter
**Contents:**
- Date of letter
- PE/RA license number and expiration date
- Applicant/Property address and Job ID
- Clear statement that amendments comply with NYC Building Code
- Specific code sections referenced
- Certification that amendments are consistent with original approved design intent (or why they're not)
- Signature in original blue ink or stamped digital signature
- Seal impression (professional license seal)

**Example Language:**
> "I hereby certify that the amended plans for [Property Address], Job ID [#], are in compliance with the 2014 NYC Building Code, particularly [sections], and represent a professional design that maintains the safety and performance standards of the originally approved application."

### Explanation/Justification Letter
**Contents:**
- Date and Job ID
- Concise explanation of each change (bullet format recommended)
- Reason for change:
  - "Field condition discovered during construction"
  - "Owner request to improve aesthetics/functionality"
  - "Inspection failure requiring code correction"
  - "Supplier unavailability necessitating material substitution"
- Code compliance statement (brief)
- Impact analysis:
  - Does change affect any other aspects of the project?
  - Will other permits be impacted?
  - Any coordination needed with other departments (FDNY, etc.)?

**Tone:** Professional, concise, honest. Avoid defensiveness. DOB examiners respond better to straightforward, clear explanations.

### Comparison/Markup Documents (Recommended but Not Always Required)
- Original approved plan pages with red-line markups showing changes
- Can be reduced-size PDFs for easy side-by-side reference
- Helps examiner quickly understand scope of change
- Not a substitute for sealed amended plans

## 6. How PAA Relates to the Original Approved Application

### Maintains Original Approval
- Original **approval number** (Job ID) remains unchanged
- Original **approval date** remains in the record
- Original **scope of work** is still the baseline
- Original **permit class/filing type** generally unchanged (unless Major PAA reclassifies)

### Creates Amendment Record
- New **PAA number/reference** is assigned in DOB NOW
- Linked to original Job ID in the system
- Creates **amendment chain** showing all approvals and amendments
- When project is completed, final Certificate of Occupancy references all amendments

### Approval Timeline
- PAA review happens **after** original approval
- Doesn't restart approval process
- Timeline is much shorter (5-30 days vs. 60-120 days for new filing)
- Original permit remains valid **during** PAA review (see Section 12)

### Relation to Permits
- Original **Work Permit** remains valid during PAA processing
- Work can **continue** while PAA is under review (within original scope)
- Once PAA is approved, Work Permit is implicitly updated to include amended scope

## 7. Professional Certification Requirements

### Who Can Certify PAA?

**Default Approach (Recommended):**
- Same **PE/RA who certified original application** can certify PAA
- Maintains design continuity and examiner familiarity
- Faster review process

**Alternative Approach:**
- **Different PE/RA can certify PAA**, but:
  - New PE/RA must explicitly adopt and endorse original design
  - Professional certification letter must state: "I have reviewed the originally approved plans and hereby adopt them as my professional work, except as noted in the amendments"
  - May trigger additional examiner questions
  - Generally not recommended unless original PE/RA is unavailable

### Certification Requirements

**All amended plans must be:**
1. **Signed** in permanent blue ink (or stamped with digital signature)
2. **Sealed** with PE/RA professional seal impression
3. **Dated** (revision date of amendment)
4. Include **license number** and **expiration date** visible on seal

**Professional Seal:**
- PE/RA license seal must be current and valid
- Seal must be legible and complete
- Some examiners require seal to show: name, license number, jurisdiction (NY)

### Can the Professional Change?

**Yes, with caveats:**
- New professional must have appropriate license for work type (PE for structural/MEP, RA for architectural)
- New professional should explicitly adopt original design and certify amendments
- May cause slight delays in review as new examiner verifies continuity
- Unusual circumstances (original professional retired, moved, etc.) are acceptable reasons for change
- Always notify original professional of transfer if contractually bound

**Best Practice:** Maintain same professional whenever possible. If change necessary, document why in explanation letter.

## 8. Common PAA Scenarios

### Scenario 1: Field Condition Changes During Construction

**Situation:** Contractor discovers structural member damaged, utilities in unexpected location, or site condition different from survey.

**PAA Approach:**
1. Obtain **field condition report** from contractor/engineer
2. Have PE develop **field change solution** (e.g., column relocation, utility reroute)
3. Prepare **amended plans** showing new configuration
4. **Certification letter** must address:
   - How field condition was discovered
   - Why original design is no longer feasible
   - How new design maintains safety/code compliance
5. **Explanation letter** describes field condition clearly
6. Mark as **Minor PAA** if straightforward field fix, **Major PAA** if complex

**Timeline:** Fast-track possible (5-7 days) if documentation is clear.

**Risk:** If field condition indicates original design was non-compliant, DOB may require additional corrections beyond the PAA scope.

---

### Scenario 2: Owner-Requested Design Changes After Approval

**Situation:** Owner wants to move a wall, change finishes, add features, or modify layouts after approval but before/during construction.

**PAA Approach:**
1. Confirm change is **not structural** and **not egress-related** (otherwise = Major PAA)
2. PE/RA prepares **amended plans** with owner-approved changes
3. **Certification letter** must state:
   - Change is owner-requested
   - Change complies with building code
   - Change does not compromise safety
4. **Explanation letter** describes aesthetic/functional improvement
5. Obtain **written owner authorization** (include with filing)
6. Mark as **Minor PAA** if layout/aesthetic, **Major PAA** if impacts systems/egress

**Timeline:** 7-10 days if Minor PAA.

**Common Changes:** Bathroom layout modifications, interior wall relocations, upgraded mechanical systems, finish material changes.

---

### Scenario 3: Addressing Inspection Failures

**Situation:** DOB or FDNY inspection identifies non-compliance requiring design correction.

**PAA Approach:**
1. **Do not proceed without addressing failure**—work permit can be revoked
2. Inspector typically issues **Notice of Violation (NOV)** or inspection report
3. PE/RA reviews failure and prepares **corrective amended plans**
4. **Certification letter** addresses:
   - Specific code section or inspection requirement failed
   - How amended design corrects failure
   - Compliance with current code
5. **Explanation letter** should be straightforward:
   - Reference inspection report date
   - Describe correction plainly
   - No need to be defensive
6. Mark as **Minor PAA** if simple code fix, **Major PAA** if systemic redesign needed

**Timeline:** 5-15 days depending on complexity.

**Tip:** Contact examiner by phone after submission if failure was critical—expedite possible with good documentation.

---

### Scenario 4: Code Compliance Adjustments

**Situation:** Code interpretation issue, referenced standard changed, or material became unavailable requiring substitution.

**PAA Approach:**
1. **Material Substitution Example:**
   - Original plans specified Product X (discontinued)
   - Equivalent Product Y meets same performance specs
   - Provide **product data sheets** showing equivalency
   - Certification letter references performance equivalency

2. **Code Interpretation Example:**
   - Original design assumed certain code interpretation
   - New interpretation requires modification
   - Certification letter clearly states:
     - Reference to specific code section
     - Citation of Building Department policy memo or memo number (if applicable)
     - How amended design complies with correct interpretation

3. **Standard Change Example:**
   - Referenced standard (e.g., ASTM, NFPA) was updated
   - New edition has different requirements
   - Amended design complies with new standard
   - Certification references both old and new standard versions

**Timeline:** 7-14 days.

**Documentation:** Include relevant code sections, policy memos, product data sheets with submission.

## 9. PAA Review Process

### Examiner Review

**Initial Review (1-3 days):**
- Examiner checks completeness of filing
- Verifies original application status
- Confirms professional seal and signatures valid
- Categorizes as Minor or Major (may upgrade from filing category)

**Technical Review (3-7 days for Minor, 7-20+ days for Major):**
- Examiner compares amended plans to original approved plans
- Verifies code compliance of changes
- Checks that PE/RA certification is appropriate
- Verifies amendments don't trigger new department coordination (FDNY, DEP, etc.)
- Reviews explanation letter for reasonableness

**Red-Flag Issues:**
- Incomplete seal/signature on amended plans
- Professional license expired
- Change is actually Major PAA (filed as Minor)
- Amended plans don't clearly show original vs. new configuration
- Certification letter is vague or non-committal
- Change appears to violate code (even if professional certified)

### Objections

**If Examiner Has Concerns:**
- DOB NOW displays **"Objections Issued"** status
- Written objections detail specific issues
- Applicant has **10-15 days** to respond [VERIFY exact timeline]
- Response options:
  1. **Submit clarification** addressing examiner concerns
  2. **Revise amended plans** and resubmit
  3. **Appeal/request supervisor review** if you disagree with objection
  4. **Withdraw PAA** and file new application (rare)

**Common Objections:**
- "Plan clarity issue—original and amended dimensions not clearly distinguished"
- "Code section [X] appears not satisfied by amendment"
- "Professional certification letter insufficient—please clarify how change maintains [system] performance"
- "Coordination with [FDNY/DEP/other] not addressed"
- "Change appears structural—should be Major PAA, not Minor"

### Approval Paths

1. **Approved Immediately** (ideal):
   - No issues identified
   - Amendment clearly complies with code
   - All documentation complete
   - Status: "Amendment Approved"

2. **Approved With Corrections** (common):
   - Minor corrections required to plans/documents
   - Examiner identifies specific items to revise
   - Applicant revises and resubmits within 10 days
   - Status changes to "Approved" upon correction acceptance

3. **Objections Requiring Response** (requires action):
   - Examiner requests clarification or code compliance justification
   - Applicant must respond substantively
   - May require plan revisions
   - Review resumes after response

## 10. Timeline for PAA Approval

### Minor PAA Timeline
- **Filing to Examiner Assigned:** 1-2 days
- **Examiner Review:** 3-7 days
- **Approval (no objections):** 5-10 days total
- **With minor corrections:** 10-15 days total
- **With objections:** 20-30 days total (including response time)

### Major PAA Timeline
- **Filing to Examiner Assigned:** 1-2 days
- **Initial Examiner Review:** 5-10 days
- **Technical Review/Objections Phase:** 10-20 days
- **Response/Correction Cycle:** 10-15 days
- **Approval (typical):** 25-45 days total
- **With substantial objections:** 45-60+ days

### Expedite Options
- **Phone contact** to assigned examiner after day 5 (polite, professional—helps with small clarifications)
- **Escalation** to supervisor if stuck beyond expected timeline
- **Re-filing** as new application if PAA review stalls (nuclear option—rarely recommended)

### What Doesn't Count as Processing Time
- Time waiting for applicant response to objections (applicant clock)
- Time for plan revisions and resubmission (applicant clock)
- Time between approval and CO issuance

## 12. Impact on Existing Permits During PAA Review

### Work Permit Validity

**Yes, Work Permit remains valid during PAA review:**
- Original **Work Permit does not expire** during PAA processing
- Work can **continue** within the scope of original approval
- Work outside original scope must **wait for PAA approval**
- If PAA is rejected, work changes cannot be implemented

### Practical Scenarios

**Scenario A: Simple Field Condition Fix**
- Original Work Permit: Approve for general foundation work
- Field condition discovered: Footing depth needs adjustment
- PAA filed for footing design change
- Work continues on other elements (framing, rough-ins, etc.)
- Once PAA approved, footing work proceeds with amended design

**Scenario B: Design Change Affecting Active Trade**
- Original Work Permit: Approve for all mechanical work
- During installation, owner wants major system redesign (Major PAA)
- Current mechanical work **pauses** (cannot proceed without amended scope)
- Work on other trades continues (electrical, plumbing, etc.)
- Once PAA approved, mechanical work resumes with new design

**Scenario C: Egress/Safety Change**
- Original Work Permit: Approve with specific egress requirements
- PAA filed to modify egress (Major PAA)
- Work **must pause** completely if egress modification affects life safety (DOB discretion)
- May require expedited PAA processing or Superintendent approval to proceed partially

### Permits from Other Agencies

- **FDNY Permit:** Not automatically updated by PAA—may require separate FDNY notification if fire safety impacted
- **CEQR/Landmarks:** If changes trigger environmental or landmark concerns, additional approvals may be needed
- **Curb Cut/Street Work:** If scope changes affect sidewalk/street, may trigger separate permits

### Certificate of Occupancy

- **Final CO is not delayed by PAA processing** (if PAA filed before CO request)
- If PAA is still under review when CO requested, DOB typically:
  - Issues **Conditional CO** excluding amended scope, OR
  - Delays CO until PAA resolved (examiner discretion)
- Once all PAAs approved, final CO is issued with reference to all amendments

## 13. Multiple PAAs on the Same Application

### Is It Allowed?
**Yes.** An application can have multiple PAAs filed sequentially or, in some cases, simultaneously.

### Sequential Multiple PAAs (Typical)

**Process:**
1. First PAA approved
2. If additional changes needed, file Second PAA
3. Each PAA references previous approval status
4. Creates amendment chain: Original Approval → PAA 1 → PAA 2, etc.

**Timeline Impact:**
- Each PAA processed independently
- Don't wait for first PAA to close before filing second (can overlap in review)
- Total project timeline extended by multiple PAA cycles

**Best Practice:** Batch changes into single PAA when possible, rather than multiple sequential filings.

### Simultaneous Multiple PAAs (Uncommon)

**When This Happens:**
- Multiple independent changes to different building systems
- Different professional certifications required
- DOB allows filing as separate PAAs with cross-references

**Risks:**
- Potential conflicts between PAAs
- Examiners may require one to be resolved before approving other
- Increases total review complexity
- Not recommended unless changes are truly independent

### Example: Sequential PAAs

1. **Original Application:** Filed and approved (Job ID #2025-001)
2. **Field Change (PAA 1):** Foundation design adjusted
   - Approved after 10 days
   - Status: Amendment Approved
3. **Owner Request (PAA 2):** Interior layout modified
   - Filed on day 8 of PAA 1 review (doesn't wait)
   - Approved 12 days after PAA 2 filing
   - Final approval chain: Original → PAA 1 → PAA 2
4. **Final CO:** References all approvals and amendments

### Impact on Original Scope

- Original application scope is baseline
- Each PAA is **additive or modifying** to prior status
- Cannot use PAA to delete approved work
- PAAs cannot contradict each other (examiner will flag)

## 15. Common Mistakes and How to Avoid Them

### Mistake 1: Filing as Minor PAA When It's Actually Major

**Problem:**
- Applicant underestimates scope of change
- Files as Minor PAA to save time/money
- Examiner upgrades to Major PAA mid-review
- Results in objections, request for additional documentation, delays

**How to Avoid:**
- When in doubt, file as Major PAA
- Err on the side of caution
- Extra fee ($1,000) is less expensive than re-filing due to incorrect classification
- Reference Section 3 for clear Major PAA indicators

---

### Mistake 2: Incomplete or Missing Professional Seal

**Problem:**
- Plans submitted without PE/RA seal impression
- Seal is smudged, illegible, or partial
- License number/expiration date not visible
- Professional signature in pencil or faded blue ink

**Result:**
- Immediate objection
- Plans returned as non-compliant
- 10-day response window to resubmit sealed plans
- Adds minimum 10 days to review timeline

**How to Avoid:**
- Have PE/RA personally stamp/seal all plan sheets
- Use **blue ink** for signatures (not black, not digital)
- Seal impression must be **clear and fully legible**
- Include PE/RA **license number and expiration date** on seal
- Verify seal is **current** (check license on NYS Department of State website)

---

### Mistake 3: Vague or Non-Committal Professional Certification

**Problem:**
- Certification letter is boilerplate/generic
- Doesn't specifically address the changes being made
- Includes disclaimer language ("subject to inspection," "if conditions permit")
- Doesn't cite specific code sections

**Result:**
- Examiner requests clarification
- Delays approval by 10-15 days
- May require phone call/meeting with professional to clarify intent

**How to Avoid:**
- Certification letter should be **specific to this PAA**
- Reference **specific code sections** (e.g., "per NYC Building Code §27-321")
- Use affirmative language: "I certify that..." not "assuming..."
- Address each change individually
- No qualifiers or conditional language

---

### Mistake 4: Poorly Marked Comparison Plans

**Problem:**
- Original and amended configurations not clearly distinguished
- Red-lines are faint, unclear, or inconsistent
- Examiner cannot easily see what changed
- Must spend extra time reviewing versus baseline

**Result:**
- Objections requesting clearer documentation
- Examiner frustration ("Please provide clearer marked-up plans")
- Extended review timeline

**How to Avoid:**
- Use **bold red lines** or bright colors for new/modified elements
- Show **original elements in light gray** with new elements in bold/color
- Include legend on every plan sheet showing what is new vs. modified
- Highlight **all changes** even if they seem minor
- Use call-outs/arrows pointing to modified areas

---

### Mistake 5: Missing or Inadequate Explanation Letter

**Problem:**
- No explanation letter submitted
- Letter is too brief ("We changed the plan")
- Doesn't explain **why** change was necessary
- Doesn't address code compliance

**Result:**
- Examiner questions reasonableness of change
- Request for more detailed justification
- Potential rejection if reasoning seems insufficient

**How to Avoid:**
- Always include detailed explanation letter
- Address:
  - **What** is being changed (specific elements)
  - **Why** it's necessary (field condition, owner request, code correction, etc.)
  - **How** it complies with code (reference specific code sections)
  - **Impact** on rest of project (other systems, other trades, etc.)
- Use clear, professional language
- 1-2 pages typical length

---

### Mistake 6: Changed Professional Mid-PAA

**Problem:**
- Original PE/RA certifies application
- Different PE/RA certifies PAA without explanation
- Examiner questions continuity of design
- May require call/meeting to clarify

**How to Avoid:**
- Use **same professional** whenever possible
- If professional must change:
  - Include explanation letter stating reason (retirement, firm closure, etc.)
  - Have **new professional explicitly adopt** original design
  - New professional's certification letter should state: "I have reviewed the originally approved plans dated [date] and hereby adopt them as my professional work..."
  - Be proactive—don't let examiner wonder about professional change

---

### Mistake 7: Filing PAA Too Late (During Active Inspection/At CO Request)

**Problem:**
- PAA filed **while DOB inspection is in progress**
- PAA filed **right before CO request**
- Work Permit status becomes ambiguous during review
- Inspection may flag non-compliance that contradicts PAA request

**Result:**
- Potential work stoppage
- Delayed CO issuance
- Inspector concerns about authority of current permit vs. pending PAA

**How to Avoid:**
- File PAA **early**, before changes are built
- Ideal timing: PAA filed within days of approval, before site mobilization
- If field change discovered: File PAA immediately upon discovery, don't delay
- Don't wait until late-stage when multiple elements are already constructed

---

### Mistake 8: Insufficient Documentation of Field Conditions

**Problem:**
- PAA is "field condition" driven
- No supporting photos, survey, or engineer report
- Examiner skeptical of claimed field change
- Hard to verify that original design truly infeasible

**Result:**
- Objection requesting "field condition documentation"
- Request for engineer field report, photos, measurements
- Adds 10-15 days while documentation is gathered

**How to Avoid:**
- When field condition discovered:
  - Photograph condition from multiple angles
  - Obtain **field report from contractor engineer** with measurements/observations
  - Include survey data or as-built measurements if relevant
  - Have site supervisor sign off on field observation
- **Submit with PAA:**
  - Photos (labeled, dated)
  - Field report excerpt
  - Engineer certification of field condition
- Examiner will quickly approve if documentation is solid

---

### Mistake 9: Overlooking Department Coordination

**Problem:**
- PAA involves fire safety change, but FDNY not notified
- PAA involves MEP system upgrade, but DEP/Con Ed not consulted
- Examiner raises objection: "Coordination with [agency] needed"
- Requires additional signoff, extends timeline

**Result:**
- 10-20 day delay for additional agency review
- PAA approval delayed until outside agency confirms no issues
- Potential rejection if outside agency has concerns

**How to Avoid:**
- Review **original approved application** for all departments listed
- If PAA affects areas coordinated by outside agencies, **proactively notify them**:
  - FDNY: Fire safety, egress, alarm systems
  - DEP: Water systems, sewage, environmental
  - Con Ed: Electrical service, transformers
  - LPC: Landmarks Preservation, historic districts
- Recommendation: Include in explanation letter: "No coordination with outside agencies required as change is [explain]"
- If coordination needed, note in submission: "FDNY has been notified—formal coordination letter attached"

---

### Mistake 10: Contradicting Original Certification

**Problem:**
- Original professional certified design was code-compliant
- PAA professional now claims original design violated code
- Creates contradiction—examiner will flag

**Result:**
- Objection: "Clarify whether original design was or was not code-compliant"
- May trigger review of original approval
- Risk of broader project review
- Timeline extended significantly

**How to Avoid:**
- Professional certification should **affirm original design was compliant**
- PAA should address **change in conditions** not **original error**:
  - Wrong: "Original design violated code section X, hereby corrected by PAA"
  - Right: "Original design was compliant with [code]. Field conditions [describe] require amended design per [code section]. Amended design maintains compliance with original intent and all applicable code."
- If original design genuinely violated code, this must be handled differently (discuss with DOB first)

## 16. Service Tiers: Standard PAA vs. Big Change PAA

### Standard PAA ($750 / 2 hours)

**Applies To:**
- Minor modifications, field fixes, simple design changes
- Material/equipment substitutions
- Non-structural layout adjustments
- Straightforward documentation, no objections expected
- Estimated processing: 5-15 days

**Characteristics:**
- Single examiner review sufficient
- Limited cross-department coordination
- Standard documentation package
- Clear, straightforward justification

**Filing:**
- Mark as **"Minor PAA"** in DOB NOW
- Fee of $750 automatically assessed
- Standard $750 service tier

---

### Big Change PAA ($1,750 / 4 hours)

**Applies To:**
- Structural modifications, egress changes, occupancy shifts
- Major system redesigns (mechanical, electrical, plumbing)
- Complex field condition resolutions
- Changes requiring coordination with multiple departments
- Estimated processing: 20-45 days

**Characteristics:**
- Multiple examiner review or specialist review
- Cross-department coordination (FDNY, DEP, etc.)
- Extensive documentation package
- More rigorous code analysis
- Higher likelihood of objections/corrections cycle

**Filing:**
- Mark as **"Major PAA"** in DOB NOW
- Fee of $1,750 automatically assessed
- Extended service tier (4 hours vs. 2 hours)
- More thorough review assumed

---

### How Service Tiers Differ in Practice

| Aspect | Standard PAA | Big Change PAA |
|--------|-------------|----------------|
| Examiner Type | Single general examiner | Specialist examiner, possibly team |
| Review Depth | Straightforward comparison | Detailed code analysis |
| Cross-Checks | Basic department list | Full coordination review |
| Typical Issues | None or minor clarification | Potential objections common |
| Objections Rate | <15% | 30-50% |
| Revision Cycles | 0-1 | 1-3+ |
| Time Investment | 2 hours | 4 hours |
| Fee | $750 | $1,750 |

---

### When to Choose Which Tier

**Choose Standard PAA ($750) if:**
- Change is clearly minor/cosmetic
- Field condition documented and straightforward
- Professional seal and certification clear
- No structural/egress implications
- No coordination with other agencies needed
- Confident documentation is complete

**Choose Big Change PAA ($1,750) if:**
- Uncertain whether change is Minor or Major
- Structural or egress-related changes involved
- Changes affect occupied spaces or life safety
- Multiple departments must coordinate
- Complex code questions
- When in doubt—choose Big Change PAA to avoid reclassification mid-review

**Best Practice:** If you have any doubt about PAA tier, file as Big Change PAA. The extra $1,000 spent upfront is far less than dealing with a reclassification objection later.
