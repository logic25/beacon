# Building Bulletin 2026-004: Software-Powered Real-Time Location Systems

**Issued:** January 21, 2026  
**Issuer:** Alan Price, P.E., Director, Office of Technical Certification & Research  
**Subject:** Real-time location software for worker safety at construction sites  
**Related Code:** BC 3301

---

## Summary

DOB approves the use of software-powered real-time location systems (smart wearables + central software) as an **option** for satisfying specific Building Code requirements related to:
1. Reporting incidents (injuries/fatalities)
2. Recording pre-shift site-safety meetings

**Key Point:** This is an OPTION, not a requirement. Traditional methods still fully acceptable.

---

## Background

### Innovation Challenge

In 2021, NYC DOB launched "Hack the Building Code Innovation Challenge" to modernize construction safety processes. Software-powered real-time location systems were among the winning technologies.

### What These Systems Do

**Technology:**
- Smart wearable devices (worn by workers)
- Central software platform
- Monitors construction activities
- Delivers safety alerts
- Enables remote recordkeeping

**NOT Covered by Building Code:**
- No existing code provisions for these systems
- No prohibition either
- This bulletin provides guidance for their use

---

## Approved Uses

### 1. Incident Reporting (Injuries & Fatalities)

**How it works:**
- Wearable device detects potential fall/incident
- System sends automatic alert
- **CRITICAL:** Responsible party must evaluate BEFORE reporting to DOB
- Verify incident is actually reportable per BC 3310.8.4

**Important Limitations:**
- System provides alerts only
- Human verification required
- Reporting responsibility unchanged
- Owner/contractor still responsible for DOB notification

**Who Must Report:**
- Entities identified in Building Code as responsible for incident reporting
- System does NOT replace this requirement
- System just helps detect incidents faster

---

### 2. Recording Pre-Shift Safety Meetings

**Traditional Requirement (BC 3301.12):**
- Competent person must maintain signed records
- Each pre-shift safety meeting documented
- Physical signatures required

**New Option with Technology:**
- System automatically registers attendance
- Creates digital record of meeting
- **Digital signature of competent person** satisfies Building Code
- Physical signatures no longer required

**Limitations:**
- Cannot SUBSTITUTE for conducting actual meetings
- Technology records attendance, doesn't replace meetings
- Competent person still required
- Same safety meeting content/requirements apply

**Clarification:**
Software-powered systems:
- ✅ CAN record attendance automatically
- ✅ CAN create digital records
- ✅ CAN use digital signatures
- ❌ CANNOT replace actual safety meetings
- ❌ CANNOT eliminate competent person requirement

---

## Additional Features (May Require Approval)

Some systems include features beyond incident reporting and safety meeting recording:
- Employee visibility/tracking
- Geofencing (location boundaries)
- Hazard proximity notifications
- Environmental sensors

**If using additional features:** Must be approved by DOB first.

---

## Critical Limitations

### What Technology DOES NOT Replace:

**Building Code Compliance:**
- Supervision requirements unchanged
- Recordkeeping obligations unchanged
- Reporting obligations unchanged
- Responsible party duties unchanged

**Example:**
- Technology detects fall
- System alerts site supervisor
- Supervisor MUST evaluate incident
- Supervisor MUST report to DOB (if reportable)
- Technology doesn't eliminate supervisor's responsibility

---

## Practical Applications for GLE Projects

### When This Matters:

**Large Projects with:**
- Multiple workers
- High-risk activities
- Complex sites
- Client wants enhanced safety monitoring

**Benefits:**
- Faster incident detection
- Automatic attendance records
- Real-time safety alerts
- Digital recordkeeping

**Costs:**
- System subscription fees
- Wearable devices
- Training
- Integration

**GLE Recommendation:**
- Optional, not required
- Consider for projects >50 workers
- Must still comply with all Building Code requirements
- Technology supplements, doesn't replace, safety practices

---

## Key Takeaways

✅ **Approved by DOB** as optional tool
✅ **Two approved uses:** Incident alerts + meeting records
✅ **Digital signatures** now acceptable
✅ **Does NOT eliminate** any Building Code requirements
✅ **Does NOT replace** human supervision/responsibility
⚠️ **Additional features** need separate approval
⚠️ **System alerts** require human verification before DOB reporting

## For Clients

**Q: "Should we use this technology on our project?"**
**A:** "It's optional. Recommended for large, complex projects with many workers. Traditional safety methods still fully acceptable. We can help evaluate if it makes sense for your project."

**Q: "Will it reduce our compliance costs?"**
**A:** "May save time on recordkeeping, but requires upfront investment in technology. Best for ongoing/large projects."

**Q: "Does it replace safety meetings?"**
**A:** "No. It just records attendance automatically. You still need to conduct actual safety meetings with a competent person."
